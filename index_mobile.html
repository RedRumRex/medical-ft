(a.) The Problem

The Edge
Cases
That Kill

Every minute in a crowded emergency department, nurses must decide who gets seen first — a wrong call can be fatal. The Emergency Severity Index provides a 1–5 scale to classify patient urgency, but even experienced clinicians can miss a critically ill patient whose vitals appear deceptively stable.

A septic patient with mild-looking numbers, an ectopic pregnancy masking as pelvic pain, or early meningitis — these are the edge cases that kill. With rising ED volumes and staff shortages, the pressure to triage quickly and accurately has never been higher.

(c.) Self-Improving Pipeline

How The
Model Gets
Betterover time

This is where the project goes beyond a demo. Every time the model runs on Hugging Face, the result JSON — containing the full triage response and reward scores — is automatically pushed back to the GitHub repository.

A daily cronjob via GitHub Actions workflow triggers train.py, which runs a GRPO-style rollout pipeline: it replays saved prompts, collects rewards, and writes new training samples to train_rollouts.jsonl. A separate script (make_dataset_from_log.py) converts these logs into structured training data.

The next time Hugging Face accepts a request, it imports the newly updated model from the GitHub repo, meaning every real interaction makes the model sharper. Each triage attempt — scored, logged, and learned from — pushes accuracy higher without any manual retraining.
